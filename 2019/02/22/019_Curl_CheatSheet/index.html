Common Usage

Demo1: Access a website or script

$ curl -4sSqL https://baidu.com
$ curl -fsSL http://somewhere/yourscript.sh | /bin/bash
$ curl -fsSL http://somewhere/yourscript.sh | /bin/bash -s parm1 parm2

-4: Use ipv4
-s: Silent mode, don’t show progress meter or error messages
-S: Show error
-q: Disable .curlrc (default settings)
-L: Follow redirects (301, 302 …)
-f: Output nothing on http errors

Demo2: Debug

$ curl -i https://baidu.com
$ curl -I https://baidu.com
$ curl -v https://baidu.com
$ curl -vv https://baidu.com
$ curl -o output.txt https://baidu.com

-i: Show response header
-I: Show response header only
-v: Show verbose
-vv: Show more verbose
-o: Write to file

Demo3: Set request methods

$ curl -X POST localhost:8080/user

-X: Set request method, [GET, POST, PUT, DELETE …]

Demo4: Set request header

$ curl \
  -H 'User-Agent: Mozilla/5.0' \
  -H 'Accept: application/json' \
  http://localhost:8080/api/v1/foobar 
$ curl -A 'Mozilla/5.0' http://localhost:8080/api/v1/foobar 
$ curl -b name=admin http://localhost:8080/api/v1/foobar 
$ curl --compressed http://localhost:8080/api/v1/foobar

-H: Set request header
-A: Equals -H ‘User-Agent:’
-b: Equals -H ‘Cookie:’
—compressed: Equals -H ‘Accept-Encoding: deflate, gzip’

Demo5: Send data

  • x-www-form-urlencoded
$ curl -d 'key1=val1&key2=val2' http://localhost:8080/api/v1/foobar
$ curl -d @data.txt http://localhost:8080/api/v1/foobar
  • json
$ curl -d '{"key": "value"}' -H 'Content-Type:application/json' \
http://localhost:8080/api/v1/foobar
  • multipart/form-data
$ curl \
-F field1=test \
-F logo.jpg=@logo.jpg \
http://localhost:8080/api/v1/foobar

Demo6: Proxy

# sock5 proxy
$ curl -x socks5h://127.0.0.1:1086 https://google.com
# http proxy
$ curl -x http://127.0.0.1:1087 https://google.com

-x: Set proxy
